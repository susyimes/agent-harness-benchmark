# TASK.md

你将接手一个存在缺陷的 Python mini-runner 项目。

## 你的目标

1. 阅读代码与测试，理解当前架构。
2. 运行现有测试，记录失败原因。
3. 修复已知缺陷并补全缺失功能。
4. 每修复一类问题，至少补一条回归测试。
5. 确保公开测试全部通过。
6. 运行 benchmark 并输出结果摘要。
7. 提交最终变更说明。

## 功能边界

项目必须支持：

- 并发执行
- timeout
- retry
- 循环依赖检测
- dry-run
- JSON 报告输出

## 额外要求

- 不引入外部服务依赖。
- 尽量保持项目结构稳定，不做无必要重写。
- 优先做最小充分修复，而不是大规模推倒重来。
- 新增测试应覆盖真实回归风险。

## 结果交付

请输出：

1. 修复清单
2. 新增测试清单
3. benchmark 结果
4. 已知限制
5. 你认为最容易被低质量 agent 框架做坏的 3 个点
